OPERATIONS DESCRIPTION

Operations determine the contents of the program's (the model's) knowledge base; they are described in a separate object (with extension .opr).

Operations object has the following format:

$Operations
<operation_description> { <operation_description> }
$End
operation_description
The operation has the following format:
<operation_name> : <pattern_name> <pattern_parameters_values>
operation_name
Operation name is a simple name. Names should be different for different operations; they should not coincide with any names determined before.
pattern_name
Pattern name is the name of one of the patterns described in pattern object.
pattern_parameters_values
Pattern parameter values are indicated in positional accordance with the order of parameters description for the pattern in pattern object. The values are determined by integer or real constants, or by value names according to the parameter type. For parameter with default values indicated, there may be symbol "*" instead of the initial value. In that case the parameter will be assigned its default value. If a range of possible values is indicated for a parameter, then it is checked up whether the initial value is within that range.

Examples

$Operations
  Operation_1 : Pattern_1      5
  Operation_2 : Pattern_1      *
  Rule_1      : Pattern_2     10    1.0
  Rule_2      : Pattern_2     10     *
$End

Remarks

Patterns have no independent value, they only have to be used for describing operations. When modeling, the simulator checks up opportunities for fulfilling the operations, not patterns. So if some pattern has been described in pattern object and never mentioned in operation object, it will never be used.


The method for writing keyboard operations slightly differs from what has just been considered. They allow to develop interactive programs in RAO language. Patterns of type keyboard are used for it. Correspondence is introduced between each keyboard operation and some key or combination of keys. Once the key is pressed, preconditions are checked up, and if they are satisfied then the event of starting the key operation is initiated. Further progress of the operation does not differ from operations of other types. Thus, the only distinction of a keyboard operation is that satisfying its preconditions determined in the pattern is not enough for starting it, pressing the corresponding key is necessary too.

A keyboard operation may also be initiated by pressing a mouse button in the animation frame's active area related to the operation. The mechanism of fulfilling a keyboard operation on pressing a mouse button does not differ from fulfilling it on pressing a key (combination of keys) on the keyboard. One keyboard operation may only correspond with one unique key combination and any amount of active areas in the animation frame.

In keyboard operation description, the key code is written after the pattern name. After pressing that key there will be an attempt to fulfil the operation. The code is followed by the pattern parameter values if there are any.

The key code may be determined by the following means. For using a digit or a letter key it is necessary to indicate the corresponding symbol in quotation marks. Letter key names are indicated with capital Latin (English) letters. For example, the following keyboard operations will be fulfilled when pressing digit 5 and letter R (provided the other preconditions are satisfied).

Keyboard_operation_1 : Keyboard_pattern '5'  12
Keyboard_operation_R : Keyboard_pattern 'R'  25

Besides letter and digit keys, some other keys may be used. These keys with their identifiers are given in the table. The identifiers should be indicated when describing keyboard operations.

Identifier Key
'ESCAPE' Exit (Escape) key
'F2' - 'F12' Functional keys F2 to F12 (F1 is used in Windows applications as Help key, so it is spared here)
'TAB' Tab key
'SHIFT' Shift key
'CONTROL' Ctrl (Control) key
'BACK' Backspace key
'RETURN' Return key (Enter)
'INSERT' Insert key (Ins)
'HOME' Home key
'PRIOR' PageUp key
'DELETE' Delete key
'END' End key
'NEXT' PageDown key
'UP' Up arrow
'LEFT' Left arrow
'DOWN' Down arrow
'RIGHT' Right arrow
'NUMPAD0' - 'NUMPAD9' Keys 0 to 9 on the digital part when Num Lock is on
'DIVIDE' Slash key on the digital part
'MULTIPLY' Times key on the digital part
'SUBTRACT' Subtraction key on the digital part
'ADD' Addition key on the digital part
'CLEAR' Key 5 on the digital part when Num Lock is off
'SPACE' Space bar
'DECIMAL' Point key on the digital part when Num Lock is on

For example, the following keyboard operations will be performed when pressing Space Bar, Right Arrow and 5 on the digital part when Num Lock is on.

Keyboard_operation_space_bar   : Keyboard_pattern 'SPACE'
Keyboard_operation_right_arrow : Keyboard_pattern 'RIGHT'
Keyboard_operation_5_digital   : Keyboard_pattern 'NUMPAD5'

All the keys just mentioned (except Shift and Control) may be used together with Shift or Control, pressed separately or simultaneously. The examples below show how to write key combinations when describing keyboard operations.

Shift_arrow      : Keyboard_pattern 'SHIFT' + 'RIGHT'
Ctrl_arrow       : Keyboard_pattern 'CONTROL' + 'RIGHT'
Shift_Ctrl_arrow : Keyboard_pattern 'SHIFT' + 'CONTROL' + 'RIGHT'

The keyboard operations described above will be fulfilled when Shift and Right Arrow are pressed together; when Ctrl and Right Arrow are pressed together; when Shift, Ctrl and Right Arrow are pressed together.

A keyboard operation may have no key code at all. In that case, word 'NOKEY' is used instead of key code when describing such operation. Operations of that kind are used when active areas of the animation frame are used for the initiation and no keys are necessary.

Each key combination may only be assigned to one keyboard operation. The following combinations used in RAO simulator for shifting simulation regimes are not allowed:

  • 'CONTROL' + 'A'
  • 'SHIFT' + 'CONTROL' + 'A'
  • 'CONTROL' + 'R'
  • 'SHIFT' + 'CONTROL' + 'R'
  • 'CONTROL' + 'M'
  • 'SHIFT' + 'CONTROL' + 'M'

Active areas initiating keyboard operations when a mouse button is pressed are described in animation frames object.
