3. Теперь надо собрать статические либы буста
3.1. Заходим в корень буста и запускаем
bootstrap.bat
Я делал это из Far'а, наверное и cmd.exe подойдет
с cmd.exe проблем также не замечено.

Если вывелось
Bootstrapping is done. To build, run:

    .\bjam
, то всё хорошо, переходим к 3.2.

Иначе смотрим на ошибку. У меня в путях (переменная Path) было с виду правильное имя
"C:\Program Files\Microsoft DirectX SDK (August 2006)\Utilities\Bin\x86"
Оно в кавычках, всё хорошо, но инсталятор упорно ругался на его концовку
\Utilities\Bin\x86";"C:\Program...
Видать ему не понравилась скобка. В общем убираем этот путь из Path хотябы на время установки буста.
Повторяем 3.1.

3.2. Из корня запускаем bjam.exe с параметрами
для компиляции дебаговских библиотек
bjam --toolset=msvc-8.0 debug link=static threading=multi

для компиляции релизных
bjam --toolset=msvc-8.0 release link=static threading=multi runtime-link=static

дело это еще более долгое, чем пункт 2, так что тоже не нервничеам;-)
Собранные либы лежат в 'C:\boost_1_43_0\stage\lib'

4. Запускаем Visual Studio, заходим в настройки
Tools/Options/Projects and Solutions/VC++ Directories

Выбираем
Platform = Win32
Show Directories for = Include Files
Под ней нажимаем на кнопку с папкой (вторая слева) и добавляем путь к инклюдам буста, в нашем случае
с:\boost_1_43_0 (да, да, корень буста)

Выбираем
Show Directories for = Library Files
Добавляем путь к скомпиленным на шаге 3 либам
C:\boost_1_43_0\stage\lib

Теперь можно компилировать rao_studio.
