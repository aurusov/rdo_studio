3. Теперь надо собрать статические либы буста
3.1. Заходим в корень буста и запускаем
bootstrap.bat
Я делал это из Far'а, наверное и cmd.exe подойдет
с cmd.exe проблем также не замечено.

Если вывелось
Bootstrapping is done. To build, run:

    .\b2
, то всё хорошо, переходим к 3.2.

Иначе смотрим на ошибку. У меня в путях (переменная PATH) было с виду правильное имя
"C:\Program Files\Microsoft DirectX SDK (August 2006)\Utilities\Bin\x86"
Оно в кавычках, и  инсталятор упорно ругался на
\Utilities\Bin\x86";"C:\Program...
Заходим в редактор переменных окружения и удаляем все двойные кавычки из PATH. Делаем это в двух местах: для пользователя и общесистемные.
Повторяем 3.1.

3.2. Из корня запускаем bjam.exe с параметрами (можно запускать b2.exe - разницы нет)

3.2.1
для компиляции дебаговских библиотек для Visual Stusio 2005 SP1
bjam --toolset=msvc-8.0 debug link=static threading=multi

для остальный версий Visual Stusio
bjam debug link=static threading=multi

3.2.2
для компиляции релизных для Visual Stusio 2005 SP1
bjam --toolset=msvc-8.0 release link=static threading=multi runtime-link=static

для остальный версий Visual Stusio
bjam release link=static threading=multi runtime-link=static

дело это еще более долгое, чем пункт 2, так что тоже не нервничеам;-)
чтобы ускорить процесс можно добавить флаг
-j N
где N - количество потоков компиляции,
для i7 пишем
bjam debug link=static threading=multi -j 8
bjam release link=static threading=multi runtime-link=static -j 8

для i5 пишем
bjam debug link=static threading=multi -j 4
bjam release link=static threading=multi runtime-link=static -j 4

Собранные либы лежат в 'C:\boost_1_47_0\stage\lib'

4. Запускаем Visual Studio, заходим в настройки
Tools/Options/Projects and Solutions/VC++ Directories

Выбираем
Platform = Win32
Show Directories for = Include Files
Под ней нажимаем на кнопку с папкой (вторая слева) и добавляем путь к инклюдам буста, в нашем случае
с:\boost_1_47_0 (да, да, корень буста)

Выбираем
Show Directories for = Library Files
Добавляем путь к скомпиленным на шаге 3 либам
C:\boost_1_47_0\stage\lib

Теперь можно компилировать rao_studio.

5. Если потребуется пересобрать буст, то

5.1 bjam --clean-all
5.2 удаляем папку lib из C:\boost_1_47_0\bin.v2
5.3 удаляем папку lib из C:\boost_1_47_0\stage

и повторяем шаг 3
